# Report Template (No-Classification, No-Ref) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a "Report" (تقرير) document type that renders a rich Arabic body onto the General Book paper with **no reference number and no classification**, signed by a full-roster **employee** signer, listed in the General Book records tagged "Report".

**Architecture:** A thin `report_service.create_report()` reuses the *render core* only — `DocxEngine.fill("Report", …)` → `_adapt_general_book`/`_pp_general_book` → `html_to_docx` → `pdf_chain` (all proven by the design-phase probe) — and persists a `Book` + `BookVersion(template_id="Report")` + `Document`, mirroring `word_book_service.finish_word_session`. A new `POST /books/reports` endpoint drives it synchronously (single-page doc; PDF conversion lenient). The shared `document_service.generate_document` core is **not** touched. The frontend reuses `_fields.json` (tile + `TemplateForm` field rendering) + existing field components, adds one new full-roster employee-picker field, and branches the submit to a `createReport` mutation.

**Tech Stack:** FastAPI (Python 3.12) + SQLAlchemy/SQLite + docxtpl/python-docx + Word COM PDF; React 19 + Vite + TS + React Query + react-hook-form + Zod.

## Global Constraints

- **Live prod checkout:** every change must be committed **and pushed to `origin/main`** or the next `mng update` overwrites it. Do the work on a branch/worktree; merge to `main` when green.
- **No DB migration:** `ref_number` stays NOT NULL + UNIQUE (reports use an internal `REPORT-{id}`); `classification_code` stays NULL; the report discriminator is `BookVersion.template_id == "Report"`.
- **No ref mark on paper:** `report.docx` has the `الرقم` block **removed**; the report path never stamps a header ref or Aztec/QR code.
- **Reuse render core, not `generate_document`:** do not add "Report" to `document_service` per-template tuples; `report_service` owns the report path.
- **Bilingual parity:** every new UI string goes in `frontend/src/locales/{en,ar}.json` with both languages; run the `i18n-rtl-reviewer` after touching bilingual surfaces.
- **Template churn:** `backend/templates/*.docx` get re-saved by the live service — commit only the intentional `report.docx`; revert incidental churn to other templates.
- **Strict gates:** `venv\Scripts\mypy.exe` (strict) and `venv\Scripts\python.exe -m pytest` (`filterwarnings=error`) must pass; frontend `pnpm -C frontend run lint`, `tsc -b --noEmit`, `vitest`.
- **API type resync:** after any backend Pydantic schema/route change, run the `/sync-api-types` flow (dump openapi → `pnpm gen:api` → typecheck) and commit `openapi.json` + `api.types.ts` together.
- **Signer resolution (exact):** `Employee.name_ar` (fallback `name_en`) for the name; `Employee.position_ar` (fallback `position`, else `""`) for the designation; signature = first `Submitter.stored_sig_path` where `Submitter.employee_id == signer_employee_id` (may be None → blank signature line).
- **Sign toggle semantics:** embed the signer's signature only when `sign is True` **and** a signature file actually resolves and exists; otherwise the `التوقيع` line renders blank for a wet signature.

**Interfaces:**
- Produces: `backend/templates/GSSG-GS_300-004_Report.docx` — a docxtpl template with tokens `{{ date }}`, `{{ recipient_name }}`, `{{ subject }}`, `{{ body }}`, `{{ manager_name }}`, `{{ manager_title }}`, `{{ manager_sig }}`; **no** `الرقم` / `{{ ref }}`; author block ordered name→title→signature under labels.

The General Book template body paragraphs (verified) are, by index:
`0 {%p if ref %}` · `1 الرقم: {{ ref }}` · `2 {%p endif %}` · `3 التاريخ: {{ date }}` · `4 السيد \ {{ recipient_name }} المحترم` · `5 تحية طيبة وبعد،` · `7 {{ '' if not subject else 'الموضوع: ' ~ subject }}` · `9 {{ body }}` · `11 للتفضل بالعلم وإجراءاتكم` · `12 (blank, CENTER)` · `13 {{ manager_sig }}` · `14 {{ manager_name }}` · `15 {{ manager_title }}` · `16 {%p if cc %}` · `17 نسخة إلى: {{ cc }}` · `18 {%p endif %}`.

Transform: delete the ref block (0,1,2); reuse blank para 12 for the closing formula; relabel/reorder the author block via **run-text swaps** (13→name, 14→title, 15→sig) so the visual order is name, title, signature.

```python
# backend/scripts/build_report_template.py
"""One-shot: derive the Report template from the General Book template.

Run once, then commit backend/templates/GSSG-GS_300-004_Report.docx:
    venv\\Scripts\\python.exe backend/scripts/build_report_template.py
"""
from __future__ import annotations

from pathlib import Path

from docx import Document
from docx.enum.text import WD_ALIGN_PARAGRAPH

SRC = Path("backend/templates/GSSG-GS_300-003_General_Book.docx")
DST = Path("backend/templates/GSSG-GS_300-004_Report.docx")


def _set_text(paragraph, text: str) -> None:
    """Replace the paragraph's text, keeping the first run's formatting."""
    runs = paragraph.runs
    if not runs:
        paragraph.add_run(text)
        return
    runs[0].text = text
    for r in runs[1:]:
        r._element.getparent().remove(r._element)


def _delete(paragraph) -> None:
    paragraph._element.getparent().remove(paragraph._element)


def main() -> None:
    doc = Document(str(SRC))

    def find(token: str):
        return next(p for p in doc.paragraphs if token in p.text)

    # 1) Remove the ref block entirely (guarantees no الرقم prints, whatever
    #    the caller passes).
    _delete(find("{%p if ref %}"))
    _delete(find("الرقم: {{ ref }}"))  # noqa: RUF001
    _delete(find("{%p endif %}"))  # the FIRST endif == the ref block's

    # 2) Closing courtesy + labelled, reordered author block. Re-find AFTER
    #    the deletions so we operate on the live tree.
    lut = find("للتفضل بالعلم وإجراءاتكم")  # noqa: RUF001
    _set_text(lut, "للتفضل بالعلم وإجراءاتكم لطفاً،،،")  # noqa: RUF001

    sig_para = find("{{ manager_sig }}")     # was first in the block (CENTER)
    name_para = find("{{ manager_name }}")   # bold 14pt
    title_para = find("{{ manager_title }}")  # bold 14pt

    # The blank CENTER paragraph directly above the signature holds the closing.
    closing = sig_para._element.getprevious()
    from docx.text.paragraph import Paragraph  # local import; avoids top clutter

    closing_para = Paragraph(closing, sig_para._parent) if closing is not None else None
    if closing_para is not None and not closing_para.text.strip():
        _set_text(closing_para, "وتفضلوا بقبول فائق الاحترام والتقدير ,,,")  # noqa: RUF001
        closing_para.alignment = WD_ALIGN_PARAGRAPH.RIGHT

    # Reorder by content swap (no element moves): top→bottom becomes name,
    # title, signature.
    for p, txt in (
        (sig_para, "الاسم: {{ manager_name }}"),      # noqa: RUF001
        (name_para, "المسمى الوظيفي: {{ manager_title }}"),  # noqa: RUF001
        (title_para, "التوقيع: {{ manager_sig }}"),   # noqa: RUF001
    ):
        _set_text(p, txt)
        p.alignment = WD_ALIGN_PARAGRAPH.RIGHT

    DST.parent.mkdir(parents=True, exist_ok=True)
    doc.save(str(DST))
    print(f"wrote {DST}")


if __name__ == "__main__":
    main()
```

```python
# backend/app/services/report_service.py
"""Create a no-classification, no-ref Report on the General Book paper,
signed at creation by a picked employee. Reuses the render core only —
NOT document_service.generate_document (see the design spec)."""
from __future__ import annotations

import uuid
from datetime import UTC, datetime
from pathlib import Path

from sqlalchemy import func, select
from sqlalchemy.orm import Session

from app.core import manager_override
from app.core.config import get_settings
from app.core.docx_engine import DocxEngine, _postprocess_general_book_footer
from app.db.models import (
    Book, BookVersion, Document, Employee, GeneralBookRecipient, Submitter, User,
)
from app.services.document_service import (
    GENERAL_BOOK_BODY_SENTINEL, _output_dir_for_admin, convert_docx_to_pdf,
)

_TEMPLATE_ID = "Report"
_GS_CATEGORY = "GS"  # same category the General Book uses → shows in that list


def _resolve_signer(db: Session, employee_id: str) -> tuple[str, str, str | None]:
    """(name, designation, signature_path|None) for the picked employee."""
    emp = db.get(Employee, employee_id)
    if emp is None:
        from app.core.errors import AppError

        raise AppError("EMPLOYEE_NOT_FOUND", f"Employee {employee_id} not found",
                       http_status=404)
    name = (emp.name_ar or emp.name_en or "").strip()
    title = (emp.position_ar or emp.position or "").strip()
    sub = db.execute(
        select(Submitter).where(Submitter.employee_id == employee_id)
    ).scalars().first()
    sig = sub.stored_sig_path if sub is not None else None
    if sig is not None:
        p = Path(sig)
        if not p.is_absolute():
            p = get_settings().data_dir / p
        sig = str(p) if p.is_file() else None
    return name, title, sig


def _resolve_recipient(db: Session, recipient_id: int | None) -> str:
    if recipient_id is None:
        return ""
    row = db.get(GeneralBookRecipient, recipient_id)
    return row.name if row is not None else ""


def create_report(
    db: Session, *, operator: User, signer_employee_id: str,
    recipient_id: int | None, subject: str, date: str | None,
    body_html: str, sign: bool = True,
) -> Book:
    name, title, sig_path = _resolve_signer(db, signer_employee_id)

    # 1) Book row with a unique internal filing id (invisible on paper).
    book = Book(
        category_id=_GS_CATEGORY,
        ref_number=f"__pending_{uuid.uuid4().hex}__",
        subject=subject,
        direction="outgoing",
        classification_code=None,
        employee_id=None,
        submitted_by_user_id=operator.id,
        approval_state="approved",
    )
    db.add(book)
    db.flush()  # assigns book.id
    book.ref_number = f"REPORT-{book.id}"
    db.flush()

    # 2) Build the render data — mirrors word_book_service MINUS ref.
    data: dict[str, object] = {
        "date": date or datetime.now().strftime("%d-%m-%Y"),
        "subject": subject,
        "body": GENERAL_BOOK_BODY_SENTINEL,
        "body_html": body_html,
        "recipient_name": _resolve_recipient(db, recipient_id),
        "cc": "",
        "submitter_g": operator.employee_id or "",  # footer = signed-in account
    }
    manager_override.apply(
        data,
        {"name_ar": name, "name_en": name, "title": title, "sig_path": sig_path},
        embed=bool(sign and sig_path),
        prefer_arabic=True,
    )

    # 3) Render onto the report paper + footer sync. No ref / Aztec stamp.
    settings = get_settings()
    out_dir = _output_dir_for_admin(_TEMPLATE_ID)
    now = datetime.now(UTC).replace(tzinfo=None)
    docx_path = out_dir / f"Report_{book.ref_number}_{now:%Y%m%d%H%M%S}.docx"
    DocxEngine(settings.templates_dir).fill(_TEMPLATE_ID, data, docx_path)
    _postprocess_general_book_footer(docx_path)
    pdf_path = convert_docx_to_pdf(docx_path)  # lenient: may return None

    # 4) Document + signed BookVersion.
    doc = Document(
        template_id=_TEMPLATE_ID, ref_number=book.ref_number,
        docx_path=str(docx_path), pdf_path=str(pdf_path) if pdf_path else None,
        submission_id=str(uuid.uuid4()), role="primary",
    )
    db.add(doc)
    db.flush()

    version_no = (db.query(func.max(BookVersion.version_no))
                  .filter(BookVersion.book_id == book.id).scalar() or 0) + 1
    version = BookVersion(
        book_id=book.id, version_no=version_no, trigger="initial",
        status="approved", template_id=_TEMPLATE_ID,
        fields={"signer_employee_id": signer_employee_id, "signed": bool(sign and sig_path)},
        created_by_user_id=operator.id, document_id=doc.id,
        signed_pdf_path=str(pdf_path) if pdf_path else None,
        signed_by_user_id=operator.id, signed_at=now,
        manager_sig_embedded=bool(sign and sig_path),
    )
    db.add(version)

    # 5) FTS corpus + commit.
    from app.services.book_service import build_search_text  # existing helper

    book.search_text = build_search_text(subject=subject, ref=book.ref_number, body="")
    db.commit()
    db.refresh(book)
    return book
```

Notes for the implementer:
- `_output_dir_for_admin`, `convert_docx_to_pdf`, `GENERAL_BOOK_BODY_SENTINEL` are imported from `document_service` (already used by `word_book_service`). If `build_search_text` lives elsewhere, follow `word_book_service.finish_word_session`'s import.
- `approval_state="approved"` is a deliberate choice: the report is final at creation and never enters the approval workflow, so nothing recomputes it. The `Report` badge (Task 8) — not the approval chip — is what distinguishes it in the list.
