Desktop / installed webapp — no pull gesture

A trackpad has inertia, so the “deliberate pull” contract can't be honored on desktop. Instead the desktop-installed app gets a small refresh button in each page toolbar that runs the same React-Query refresh and shows the same ring spinner inline — shared vocabulary, different trigger. Touchscreen laptops still get the pull.

How this maps to the real app: the sheet you're dragging is the per-page scroll container (the app shell keeps <main> at overflow-hidden and each page scrolls internally), so this becomes a reusable <PullToRefresh> wrapper pages opt into. Refresh = queryClient.invalidateQueries({ type: 'active' }) for the current route, with a 500 ms minimum spinner and 8 s ceiling. Physics shown here are the real proposed numbers: dead-zone 24 px, rubber-band c=0.42, arm at 112 px visual (~170 px of finger travel), 120 ms hold-to-arm, spring settle (stiffness 260 / damping 26).
